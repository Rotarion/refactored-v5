# Advisor JavaScript Operator Contract

This document freezes the current AHK-facing contract for `assets/js/advisor_quote/ops_result.js`.
It is a contract inventory, not a refactor design.

## Runtime Contract

- Runtime file: `assets/js/advisor_quote/ops_result.js`
- AHK template placeholders: `@@OP@@` and `@@ARGS@@`
- AHK rendering path: `workflows/advisor_quote_workflow.ahk` -> `AdvisorQuoteRunOp()` -> `AdvisorQuoteRunJsOp()` -> `AdvisorQuoteRenderOpJs()`
- DevTools bridge path: `adapters/devtools_bridge.ahk` -> `RunDevToolsJsAssetWork()` -> `RenderJsTemplate()` -> `RunDevToolsJSGetResult()`
- Return channel: the operator wraps execution in `copy(String((() => { ... })()))`
- Unknown top-level op: returns an empty string
- Thrown top-level error: returns key=value lines including `result=ERROR`, `op=`, `message=`, `stack=`, and `url=`
- `@@OP@@` is rendered as a JS literal string. `@@ARGS@@` is rendered as a JS object literal by `JsLiteral()`.
- Phase 1A source scaffold: `assets/js/advisor_quote/ops_result.js` remains the injected runtime file and is generated byte-for-byte from `assets/js/advisor_quote/src/operator.template.js` by `assets/js/advisor_quote/build_operator.js`.
- Phase 1B source layout: pure text/output snippets now live under `assets/js/advisor_quote/src/core/` and are inlined by the build script; the generated runtime contract is unchanged.
- Phase 1C source layout: pure matcher/scoring snippets now live under `assets/js/advisor_quote/src/matchers/` and are inlined by the build script; the generated runtime contract is unchanged.
- Phase 1D-A source layout: foundational DOM read/support, arg-normalization, and alert-read snippets now live under `assets/js/advisor_quote/src/core/` and are inlined by the build script; the generated runtime contract is unchanged.
- Phase 1D-B1 source layout: native field value/event snippets now live under `assets/js/advisor_quote/src/core/fields.js` and are inlined by the build script; focused setters remain in `operator.template.js`; the generated runtime contract is unchanged.
- Phase 1D-B2 source layout: generic click/target interaction snippets now live under `assets/js/advisor_quote/src/core/click.js` and are inlined by the build script; the generated runtime contract is unchanged.
- Early-flow patch note: `product_overview_tile_status` was added as a read-only key=value status op so AHK can verify Product Tile Grid Auto selection before clicking Save & Continue. `click_product_overview_tile` keeps its raw string return contract but is now idempotent: if the tile is already selected, it returns `OK` without clicking.
- Product Auto gate note: `ensure_product_overview_tile_selected` was added as a key=value diagnostic/action op that never clicks an already-selected Auto tile and returns `VERIFY_FAILED` if a click does not produce scoped selected-state evidence. `click_product_overview_subnav_from_rapport` was added for the one-shot RAPPORT -> top `SELECT PRODUCT` recovery path; it excludes sidebar/Add Product controls.
- Product Tile resolver note: Product Tile status, click, and ensure ops share the same Patch A3-style resolver. The resolver starts from visible Auto text, resolves a single-product tile/card container, rejects broad product grids containing multiple labels, and then targets a real interactive descendant or the tile/card center. Auto does not need to appear in the generic buttons array.
- Patch B1/B2 note: `customer_summary_overview_status` was added as a read-only key=value diagnostic op so AHK can recognize Customer Summary / Prefill Gate when raw `detect_state` reports `ADVISOR_OTHER`; `click_customer_summary_start_here` was added as a scoped START HERE action for the Prefill Gate.
- Gather Data vehicle-add note: `gather_vehicle_row_status` was added as a read-only key=value diagnostic op. `prepare_vehicle_row` still returns only an index string or `-1`, but can now open an Add Car or Truck row before returning the usable row index.
- Gather Data year-cascade note: `set_vehicle_year_and_wait_manufacturer` was added as a bounded row-specific write/status op so AHK can force and verify the ModelYear -> Manufacturer cascade before selecting make.
- Gather Data continuation note: `gather_vehicle_add_status` was added as a read-only vehicle completion status op, and `ensure_start_quoting_auto_checkbox` was added to select the stable Start Quoting Auto checkbox before Add Product/Create Quotes actions.
- Gather Data edit-modal note: `gather_vehicle_edit_status` and `handle_vehicle_edit_modal` were added for Edit Vehicle panels that require Sub-Model before confirmation can finish. The handler may select a VIN-pattern, trim-match, or first-valid non-placeholder Sub-Model, then clicks Update only after selection verifies.
- Address Verification note: `address_verification_status` and `handle_address_verification` were added for the Create New Prospect Address Verification intermediate state. The resolver targets `snaOption` radios and `Continue with Selected`; it does not use the lower Create New Prospect button while Address Verification choices exist.
- Catalog-aware confirmed-card note: `gather_vehicle_add_status` and `gather_confirmed_vehicles_status` accept optional allowed Advisor make labels on vehicle args for confirmed-card matching. Result fields and result values are unchanged.
- ASCPRODUCT reconciliation note: the Drivers and Vehicles stage now uses `asc_*` status/action ops for participant marital/spouse truth, driver row add/remove, vehicle row add/promote, and save gating. ASCPRODUCT route detection is dynamic (`/apps/ASCPRODUCT/<id>/`) and does not depend on fixed route ids such as 109 or 112.

## Wait Condition Names

The `wait_condition` op dispatches on `args.name`.

Covered by the smoke test: all known `wait_condition` names now have direct offline positive/negative assertions, including `post_prospect_submit`, `prospect_form_ready`, `duplicate_to_next`, `gather_data`, `on_customer_summary_overview`, `on_product_overview`, `to_select_product`, `gather_start_quoting_transition`, `vehicle_select_enabled`, `vehicle_added_tile`, `vehicle_confirmed`, `on_select_product`, `select_product_to_consumer`, `consumer_reports_ready`, `drivers_or_incidents`, `after_driver_vehicle_continue`, `add_asset_modal_closed`, `continue_enabled`, `incidents_done`, `quote_landing`, `is_duplicate`, `is_customer_summary_overview`, `is_product_overview`, `is_rapport`, `is_select_product`, `is_asc`, and `is_incidents`.

Negative-evidence risk: `add_asset_modal_closed` returns `1` when `document.getElementById(args.addAssetSaveId)` is absent. That can succeed because the modal is truly closed, because the id changed, or because the DOM fixture/page is incomplete.

## Return Shape Details Proven by Smoke

- Unknown op returns `''`.
- Top-level thrown errors return key=value lines with `result=ERROR`.
- `prospect_form_status`, `address_verification_status`, `handle_address_verification`, `product_overview_tile_status`, `ensure_product_overview_tile_selected`, `click_product_overview_subnav_from_rapport`, `customer_summary_overview_status`, `click_customer_summary_start_here`, `fill_gather_defaults`, `gather_defaults_status`, `gather_vehicle_row_status`, `set_vehicle_year_and_wait_manufacturer`, `gather_vehicle_add_status`, `gather_vehicle_edit_status`, `handle_vehicle_edit_modal`, `gather_confirmed_vehicles_status`, `gather_start_quoting_status`, `ensure_start_quoting_auto_checkbox`, `ensure_auto_start_quoting_state`, `set_select_product_defaults`, `select_product_status`, `fill_participant_modal`, and `fill_vehicle_modal` return key=value blocks.
- `scan_current_page` returns JSON.
- Generic click/read ops preserve current raw string contracts: `OK`, `NO`, `NO_BUTTON`, `DISABLED`, `CLICK_FAILED`, `NO_REASON`, `NO_CONTINUE`.
- Vehicle dropdown args are currently `index`, `fieldName`, and `wantedText`; optional `allowFirstNonEmpty` is used for submodel fallback. Year selection requires an exact year. Make/model matching tries exact value/text first, then normalized exact/explicit small aliases, then unambiguous contains.
- Participant modal args are currently `ageFirstLicensed`, `email`, `military`, `violations`, `defensiveDriving`, `propertyOwnership`, and related spouse/gender args.
- `fill_vehicle_modal` currently returns keys such as `garagingAddressSameAsOtherClicked`, `purchaseDateFalseClicked`, `ownershipClicked`, and `detectedYear`.
- `customer_summary_overview_status` returns keys: `result`, `runtimeState`, `confidence`, `urlMatched`, `overviewMatched`, `startHereMatched`, `quoteHistoryMatched`, `assetsDetailsMatched`, `summaryAnchorMatched`, `startHereCount`, `evidence`, `missing`, and `url`. High confidence requires customer-summary overview URL plus START HERE plus a summary anchor such as `Quote History`, `Assets Details`, `Contact Information`, `Family Members`, or `Net Worth`; medium confidence is URL plus START HERE without those summary anchors and still sets `runtimeState=CUSTOMER_SUMMARY_OVERVIEW`; low confidence is URL-only route evidence; non-customer-summary URL is `NOT_DETECTED`.
- `click_customer_summary_start_here` returns keys: `result`, `clicked`, `targetText`, `targetTag`, `targetClass`, `urlBefore`, and `evidence`. It first verifies the Customer Summary overview route, selects only visible START HERE controls, excludes Add Product, prefers a visible button/input over a sidebar link, and never direct-navigates.

A matching Cars and Trucks / CONFIRMED VEHICLES card is primary `ADDED` evidence. Potential-vehicle text and different confirmed vehicles are not `ADDED`. Row disappearance without confirmed vehicle text is not `ADDED`. Optional vehicle args `allowedMakeLabels`, `makeAliases`, or `advisorMakeLabels` can provide `|`/`,`/`;` separated Advisor make labels for confirmed-card matching; `strictModelMatch=1` keeps model matching exact-normalized for catalog-aware confirmed cards.
